Everything Running Inside index.js

A quick-reference documentation hub covering moderation, security, AI utilities, onboarding systems, automation helpers, logging, and the persistent configuration stored in data.json.

Overview

Multi-purpose Discord bot with one core runtime

Main runtime file: index.js. Persistent config and state: data.json. Slash commands are registered per guild and the bot depends on discord.js plus @google/generative-ai.

50+ Documented commands
7 Major bot systems
1 Single-file runtime

Staff command toolkit

  • /warn, /warnings, /clear, /purgeuser
  • /lock, /unlock, /setnick, /slowmode
Security / Automod

Protective systems

Logging and Audit

What gets logged when /setlog is configured

Bans, kicks, mutes, unmutes, warnings, clear and purge actions, lock and unlock actions, unbans, slowmode updates, anti-spam actions, blocked links, message deletes, message edits, member joins and leaves, kicks detected from audit logs, role create/delete/update, ban add/remove, timeout updates, anti-raid alerts, and anti-nuke role stripping.

Stored Config

Important data.json entries

welcomeChannelId, logChannelId, warnings, antilinkEnabled, antilinkBypassChannels, antinukeEnabled, antiraidEnabled, antispamEnabled, autoRoleIds, reactionRoles, counting, aiChannelId, aiContext, aiAutoModEnabled, autoTagChannels, weeklyRecapChannelId, lastRecapDate, ticketConfig, birthdays, reminders, and global AI runtime stats under _global.aiStatus.

AI quota note

AI-powered features depend on Gemini availability and free-tier quota. When quota is exhausted, AI chat, summaries, translations, recaps, and toxicity analysis may slow down, enter cooldown, or temporarily stop until quota resets.

Free quota behavior

Some AI features are limited by free quota. If the quota is reached, the bot can fall back, cool down requests, or temporarily disable AI responses until service becomes available again.
